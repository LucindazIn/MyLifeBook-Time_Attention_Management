# 离线内容池与随机展示（替代 Gemini 扩展能力）

## 一、背景与已删除代码

以下符号已从 `[src/lib/gemini.ts](../../src/lib/gemini.ts)` 移除，不再发起对应 Gemini 调用：


| 已删除                      | 原用途                           |
| ------------------------ | ----------------------------- |
| `canCallGemini`          | `canCallFreeTierGemini` 的废弃别名 |
| `generateAgenda`         | 上传文档 → JSON 日程                |
| `generateQuote`          | 按 prompt 生成单条金句 JSON          |
| `generateDailySummary`   | 当日「意义」长文总结                    |
| `generateRandomSchedule` | chill/productive 全日 JSON 排程   |


**保留的 Gemini 能力**（仍在产品中）：`generateDayName`、`generateSummaryByFilter`，以及章节/叙事相关调用链（若在其他模块）。

## 二、参照范式：随机 Productive / Chill 日

当前实现已证明「无 LLM、可 ship」的路径：

- **内容池**：`[src/data/schedule_activities_zh.json](../../src/data/schedule_activities_zh.json)` / `schedule_activities_en.json`（或同类 catalog）。
- **组合逻辑**：`[src/lib/scheduleComposer.ts](../../src/lib/scheduleComposer.ts)` 中 `composeRandomDaySchedule`（按 mode、强度、时间窗约束）。
- **入口封装**：`[src/lib/scheduleLocalSuggestions.ts](../../src/lib/scheduleLocalSuggestions.ts)` `getLocalScheduleSuggestions(mode, language, { seed })`。

后续新能力应优先：**静态或 DB 表驱动的选项表 + 纯函数 pick（可带 `seed` 保证同日可复现）**。

## 三、能力映射（产品方向）

### 1. 随机全日排程（原 `generateRandomSchedule`）

- **状态**：产品侧应已走本地 catalog（见第二节）；删除的仅为未接线或重复的 Gemini 版本。
- **后续**：若仍有入口调用旧 API，改为只调 `getLocalScheduleSuggestions`；无需新表即可闭环。

### 2. 金句 / 一日一句（原 `generateQuote`）

- **方向**：与 `SurpriseWidgets` 中按 theme 分区的本地 `QUOTES` 一致，扩展为：
  - **V1**：继续用仓库内 JSON（按 `AppTheme` × 语言分区），条目可增多、可版本化。
  - **V2（可选）**：Supabase 表 `content_snippets`（`kind: 'quote'`, `locale`, `theme`, `text`, `author`, `weight`），客户端缓存后随机；或与 `user_collection_state` 扩展 payload 同步「用户自定义金句」子集。

### 3. 日意义短评（原 `generateDailySummary`）

- **方向**：不做长文 LLM；改为 **模板槽位 + 短句池**，例如：
  - 输入：当日是否有事件 / 是否有 meaning / 高光数量区间 / 日记是否非空（离散特征）。
  - 输出：从池子中选 1～3 句拼接，或单句「今日锚点」。
- **实现**：可先 `src/data/daily_meaning_fragments_zh.json` 等 + `pickFragment(features, seed)`；再上 DB 便于运营增删文案。

### 4. 文档 → 日程（原 `generateAgenda`）

- **方向**：若仍需要「从文件导入」：
  - **简化**：仅支持固定格式（CSV/ICS/纯文本约定），用解析器而非 LLM。
  - **或**：明确放弃该入口，避免维护成本。

## 四、数据与同步（可选）

- **仅本地**：JSON 资源 + 版本号字段即可。
- **跨设备**：在现有 `user_collection_state.payload` 或独立表中存「用户自定义片段」；原则与角色目录、标签同步一致（见 `[supabase/deploy_cross_device_sync.sql](../../supabase/deploy_cross_device_sync.sql)` 注释）。

## 五、验收建议

- 全站无对已删符号的引用；`npx tsc --noEmit` 通过。
- 各 UI 入口：无 Key / 无配额时行为与「随机日」一致（不依赖 Gemini）。
- 文案：**简体中文 / English** 分池，遵守项目 Title Case 英文规范。
