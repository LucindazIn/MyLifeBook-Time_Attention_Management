# 章节周期附带统计图 + 弹窗内「本期状态数据」（方案）

## 产品交互（新增，优先）

- 用户在 **[章节叙事](src/components/ChapterNarrativeCard.tsx)** 选好周期并点击 **「生成本章节」**，打开 **[ChapterViewModal](src/components/ChapterViewModal.tsx)** 后，应在**同一弹窗内、章节内容（叙事正文）下方**立即出现新板块，标题为 **「本期状态数据」**（英文界面可用 **"Period Stats"** 或 **"This Period At A Glance"**，与全站 Title Case 一致）。
- 该板块内嵌三块与业务含义对应的内容（顺序建议与合集页一致）：
  1. **角色能量**（周期内角色占比条 + 图例）
  2. **事件标签分析**（周期内标签分布，可与现饼图/列表一致）
  3. **生命能量**（周期内折线/点，只读、无导出按钮）
- **数据范围**必须与弹窗章节一致：使用 `chapter.periodStart` / `chapter.periodEnd`（草稿态同样），**不得**复用 CollectionView 里三张卡当前 UI 上的时间选择。

## 现状（代码依据）

- 当前弹窗主要展示标题、周期标签、叙事正文与导出/复制；**无**统计图。[`ChapterNarrativeCard`](src/components/ChapterNarrativeCard.tsx) 已持有 `events`、`completedInstances`、`dayTags`、`dayNames`、`dayVibes` 等，可通过 **新增 props** 传入 `ChapterViewModal`，供「本期状态数据」计算使用。
- 合集页三张卡片时间范围与章节周期**不对齐**（见原方案）；**弹窗内板块必须按章节日期重算**。

## 实现要点

### 1. 按章节周期聚合（与 UI 卡片解耦）

- 同原方案：**角色能量**用 [`aggregateByRole`](src/lib/roleAggregation.ts) + `expandRecurringEvents` 限定在 `[periodStart, periodEnd]`；**事件标签**抽 `computeEventTagSlicesForRange`；**生命能量**用 [`computeDailyEnergySeries`](src/lib/lifeEnergy.ts) + 只读折线组件。生命能量若依赖 [`getLifeEnergyForDay`](src/lib/lifeEnergyStorage.ts) 的 rangeKey，需定义与「仅章节周期」一致的 storage key（例如 `custom_${start}_${end}` 或与现有 `RoleBalanceCard` 规则对齐），避免读到错误区间的覆盖值。

### 2. UI 挂载位置

- 在 [`ChapterViewModal`](src/components/ChapterViewModal.tsx) 中，在叙事正文区域（`renderSummaryWithMarkdown` / 编辑区）**之后**、底部操作按钮（保存/导出等）**之前**，插入可折叠或固定高度的 **「本期状态数据」** 区块；**打开弹窗即有数据**，不依赖用户再点一步。

### 3. 可选：导出 ZIP

- 若仍需要「导出本期数据」附带图片：可 **复用同一套子组件 DOM**（或同一 ref 容器）做 `html-to-image` + JSZip；优先级低于弹窗内展示。

### 4. 验收标准

- 打开「生成本章节」后，**无需滚动到不可见区域才出现**（若内容过长可整块在正文下，自然滚动即可）。
- 切换章节周期再打开，三块数据随 `periodStart`/`periodEnd` 变化。
- 与「仅文本导出」中的周期头信息一致。

## 风险与说明

- 极长自定义周期：折线图需 **x 轴抽样** 或横向滚动，避免弹窗过宽；可与 [`EnergyLineChart`](src/components/RoleBalanceCard.tsx) 行为对齐。
- `SavedChapter` 可不存图片；板块为**即时计算渲染**，落库仍仅为文本章节。

## 结论

**可以实现**；**首要交付**为弹窗内 **「本期状态数据」** 区块；导出 ZIP 为可选增强。
